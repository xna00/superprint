## 开发建议

### 认证和权限
1. 使用正确的 secret 获取 access_token
2. 注意不同应用的权限范围
3. 第三方应用需注意权限申请

### 回调处理
1. 正确配置回调 URL
2. 实现消息解密和验证
3. 及时响应回调请求

### 性能优化
1. 缓存 access_token（有效期 2 小时）
2. 合理使用批量接口
3. 避免频繁调用限流接口

### 安全建议
1. 不要在前端暴露 secret
2. 验证回调消息的签名
3. 使用 HTTPS 协议

## 常见问题

### Q: access_token 获取失败?
A: 检查 corpid 和 secret 是否正确，确认应用是否启用。

### Q: 回调验证失败?
A: 确认回调 URL 可访问，检查签名算法和 token 配置。

### Q: 权限不足错误?
A: 检查应用是否有对应接口的调用权限，查看应用可见范围设置。

### Q: 用户 ID 转换问题?
A: 企业微信已升级 ID 安全机制，注意使用正确的 ID 转换接口。
